# Security Policy

## Supported Versions

PythonTools is currently under active development.  
Security updates will be applied to the latest release branch:

| Version | Supported |
|---------|-----------|
| 0.x.x   | ✔ Active development |
| <0.x.x  | ✖ Not supported |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### Contact

**security@linktechengineering.net**

Please include:

- A clear description of the issue  
- Steps to reproduce (if applicable)  
- Potential impact  
- Any suggested fixes  

You will receive an acknowledgment within 48 hours.

## Disclosure Policy

- Vulnerabilities will be evaluated promptly  
- Fixes will be prioritized based on severity  
- Public disclosure will occur only after a patch is available  
- Credit will be given to reporters unless anonymity is requested  

## Security Expectations

Contributors should:

- Avoid introducing unnecessary dependencies  
- Prefer explicit, deterministic logic  
- Validate inputs where appropriate  
- Avoid project‑specific assumptions in shared modules  
- Flag any suspicious behavior or unexpected side effects  

## Scope

This policy applies to:

- PythonTools source code  
- Packaged releases  
- Documentation and examples  

It does *not* apply to downstream projects (e.g., RunUpdates), which maintain their own security policies.
